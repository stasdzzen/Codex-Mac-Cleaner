# Политика безопасности

## Поддерживаемые версии

| Версия | Security fixes |
|---|---|
| Последний опубликованный Release | Да, после появления первого Release |
| Более ранние alpha/beta Releases | Не гарантируются |
| Source-код ветки `main` | Не считается установленным плагином |

Пока Release отсутствует, политика применяется к исходному коду, MCP-контрактам,
plugin manifests, quarantine/policy logic и repository supply chain.

## Как сообщить об уязвимости

Не создавайте публичную Issue и не раскрывайте рабочий exploit. Используйте
только [GitHub Private vulnerability reporting](https://github.com/stasdzzen/Codex-Mac-Cleaner/security/advisories/new).

В закрытом отчёте укажите:

* затронутый component и commit либо version;
* возможный impact;
* minimal reproduction на синтетических данных;
* ожидаемое и фактическое поведение;
* безопасное доказательство без secrets, tokens и персональных данных.

Не прикладывайте содержимое чужого `HOME`, реальные пути, полный environment
dump, app inventory или данные, не нужные для воспроизведения.

## Что происходит после отчёта

Сопровождающие проверят воспроизводимость и влияние в закрытом канале. Если
проблема подтверждена, исправление, release и disclosure координируются до
публичного раскрытия. Срок ответа зависит от влияния и доступности
сопровождающих; публичный SLA не предоставляется.

Архитектурные границы безопасности описаны в
[модели безопасности](docs/safety/safety-model.md) и
[модели угроз](docs/safety/threat-model.md).
