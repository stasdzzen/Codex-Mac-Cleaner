# Поддержка

Codex Mac Cleaner находится на ранней alpha-стадии. Поддержка предоставляется
по мере доступности сопровождающих; проект не гарантирует SLA, срочную
индивидуальную помощь или сопровождение production-систем.

## Куда обращаться

| Запрос | Канал |
|---|---|
| Воспроизводимая ошибка | [Bug Issue Form](https://github.com/stasdzzen/Codex-Mac-Cleaner/issues/new?template=bug.yml) |
| Ограниченное улучшение | [Feature Issue Form](https://github.com/stasdzzen/Codex-Mac-Cleaner/issues/new?template=feature.yml) |
| Вопрос об использовании | Этот документ и [существующие Issues](https://github.com/stasdzzen/Codex-Mac-Cleaner/issues) |
| Уязвимость | [Private vulnerability reporting](https://github.com/stasdzzen/Codex-Mac-Cleaner/security/advisories/new) |

Перед новой Issue проверьте, что похожее обращение ещё не создано. Общую идею
сначала сузьте до наблюдаемой проблемы и проверяемого результата.

## Безопасная диагностика

В обращении укажите:

* commit или release version;
* версии macOS и Codex, модель Mac и архитектуру;
* профиль аудита;
* ожидаемое и фактическое поведение;
* минимальные шаги воспроизведения на синтетических данных;
* только относящийся к проблеме фрагмент ошибки.

Не публикуйте tokens, credentials, subscription URLs, персональные данные,
реальные полные пути, app inventory и полные environment dumps. Уязвимости не
диагностируются в публичной Issue.
