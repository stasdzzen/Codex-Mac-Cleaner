# Безопасное обновление установленного плагина

Codex CLI хранит Git marketplace как snapshot с закреплённым ref. Команда
`codex plugin marketplace upgrade` обновляет содержимое только в пределах уже
заданного ref и не переключает beta.2 на beta.3. Для перехода между
неизменяемыми release-тегами нужно заменить именно marketplace snapshot, но
удалять запись установленного плагина не требуется.

Публичный пакет содержит отдельный навык `codex-mac-cleaner-update` и команду
`scripts/codex-mac-cleaner-update.mjs`. Они активируются только явным запросом
с точным тегом. Команда:

1. читает структурированное состояние через Codex CLI;
2. требует установленный и включённый `codex-mac-cleaner` из канонического Git
   repository;
3. проверяет существование точного удалённого release-тега до изменений;
4. удаляет только старый marketplace snapshot, добавляет канонический source с
   новым ref и повторно выполняет `plugin add`;
5. проверяет полученную версию;
6. при ошибке после начала изменений восстанавливает `v<предыдущая версия>` и
   повторно проверяет установленный плагин.

Updater не принимает repository URL или marketplace name от пользователя, не
выбирает `latest`, не редактирует конфигурацию Codex напрямую и никогда не
вызывает `plugin remove`. Он не входит в MCP runtime очистки, не получает пути
аудита и не касается пользовательских данных. Единственное сетевое обращение
относится к уже утверждённой supply-chain границе GitHub Releases/marketplace из
[ADR-0007](../decisions/ADR-0007-github-distribution.md): наружу передаются
только канонический repository и точный ref. Основной аудит и mutation runtime
остаются полностью локальными. После успешного обновления нужен
полный перезапуск Codex и новая задача, потому что текущий процесс уже загрузил
старый набор навыков и MCP-инструментов.

Если rollback не подтверждён, результат считается блокирующим. Нельзя
предлагать `sudo`, ручное изменение конфигурации или продолжать работу с
непроверенным состоянием.
