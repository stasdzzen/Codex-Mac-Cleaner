# Исходная идея

Codex Mac Cleaner — локальный open-source плагин для Codex. Он исследует Mac по запросу пользователя, находит вероятные остатки удалённых приложений и показывает доказательства для каждой находки.

Продукт не обещает «оптимизировать» систему одной кнопкой. Он должен ответить на четыре вопроса:

1. Откуда появился объект.
2. Почему объект признан кандидатом на очистку.
3. Что может перестать работать после перемещения.
4. Как вернуть объект на прежнее место.

# Исходный сценарий

1. Пользователь запускает read-only аудит из чата.
2. Локальный MCP-сервер собирает наблюдения из разрешённых областей macOS.
3. MCP App показывает сводку, таблицу находок и доказательства.
4. Пользователь отдельно подтверждает перемещение каждого разрешённого объекта.
5. Сервер перемещает объект в локальный карантин и записывает журнал.
6. Пользователь восстанавливает объект либо вручную очищает карантин после дополнительной проверки.

# Что уточнила архитектурная сессия

Утверждённые решения находятся в [границах MVP](scope-and-principles.md) и [ADR](../decisions/). Они имеют приоритет над этим архивным снимком.

Главные уточнения:

* минимальная платформа — macOS 26, только `arm64`;
* developer cleanup исключён из v0.1;
* Full Disk Access не обязателен;
* Swift-helper, `sudo`, фоновые демоны и автоматическая очистка отсутствуют;
* документация хранится как OKF bundle;
* реализация распространяется через GitHub Releases и repo marketplace.
