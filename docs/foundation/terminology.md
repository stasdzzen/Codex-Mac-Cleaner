# Глоссарий

| Термин | Значение | Не использовать |
|---|---|---|
| Аудит | Read-only сбор и классификация наблюдений | Очистка, лечение |
| Находка | Объект, включённый в отчёт аудита | Мусор без доказательств |
| Доказательство | Структурированное наблюдение из конкретного источника | Мнение модели |
| Метка | Состояние находки: активна, простаивает, осиротела, дубликат или неизвестно | Вердикт на удаление |
| Уверенность | Высокая, средняя или низкая воспроизводимость классификации | Вероятность безопасности |
| Разрешённые действия | Результат policy engine для текущей ревизии аудита | Решение интерфейса |
| Уровень поддержки | `candidate`, `analysis_only` или `unsupported_manual` | Степень уверенности |
| Защищённая область | Встроенное server-side исключение из кандидатов или mutation | Пользовательский UI-фильтр |
| Безопасные метаданные | Формат, статус разбора, размер, время, безопасный владелец и sensitivity flags без сырых значений | Содержимое конфигурации |
| Карантин | Управляемое локальное хранилище с исходным путём и восстановлением | Корзина, удаление |
| Восстановить | Вернуть payload в исходный путь | Отменить удаление |
| Очистить карантин | Необратимо удалить отдельно подтверждённый payload | Очистить всё |
| Покрытие | Области и источники, которые аудит смог или не смог проверить | Полнота диска |
| Ревизия аудита | Неизменяемая версия результатов и fingerprint | Текущий экран |
| Отменённый аудит | Неполный read-only отчёт без mutation-действий | Завершённая ревизия |
| Логический размер находок | Сумма логических размеров находок текущей ревизии | Можно очистить |
| Физический размер находок | Сумма физически занятых байтов находок текущей ревизии | Точно освободится |
| В карантине | Physical size активных payload; место ещё занято | Освобождено |
| Удалено навсегда | Physical size записей `purged` в локальном журнале | Точно освобождено на APFS |
| Свободно на диске | Наблюдение `statfs` с временем измерения | Результат последнего purge |
| Оценка освобождения | Snapshot-оценка physical bytes с confidence, basis и limitations | Гарантированное изменение `df` |
| Удалить | После отдельного подтверждения переместить один исходный объект в карантин | Direct delete исходного объекта |
| Исключить | Сохранить локальное identity-based правило и не предлагать совпавший объект в следующих аудитах | Ослабить protected scope или разрешить mutation |
| Пропустить сейчас | Пометить находку просмотренной в текущем UI-сеансе без файловой операции | Постоянно игнорировать |
| Расписание | Явно включённая Codex automation для ежемесячного read-only аудита | Cron, LaunchAgent, фоновая очистка |
| Preview token | Одноразовое краткоживущее разрешение на конкретное действие | Подтверждение модели |

# Тон интерфейса

Интерфейс говорит нейтрально и конкретно. Ошибка объясняет, что произошло и что делать дальше. Кнопки называют действие: «Начать аудит», «Отменить аудит», «Показать в Finder», «Переместить в карантин», «Исключить», «Пропустить сейчас», «Снова проверять», «Восстановить», «Удалить навсегда», «Включить ежемесячную проверку», «Настроить», «Не сейчас», «Отмена».

Формулировки «Да», «Нет», «Оптимизировать», «Ускорить Mac», «Безопасно удалить», «Очистить всё» и готовые shell-команды для исправления запрещены.
