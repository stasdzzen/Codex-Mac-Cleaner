# Качество

Проверки, без которых архитектурные обещания не считаются реализованными.

## Концепты

* [Стратегия тестирования](test-strategy.md) — уровни, fixtures и fault injection.
* [Критерии приёмки](acceptance-gates.md) — блокирующие условия для PR и release.

## Принцип

Отчёт агента или зелёный unit test не заменяет проверку destructive-flow. Completion claim требует свежего вывода обязательных gates и отдельно отмеченного real-Mac smoke.
