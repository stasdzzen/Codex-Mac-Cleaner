# Codex Mac Cleaner v0.1.0-beta.10

Версия beta.10 уточняет окно проверки, позволяет большому аудиту обработать все
найденные объекты без общего ограничения по времени и обновляет две зависимости
среды выполнения до проверенных безопасных версий.

## Что изменилось

- Главное окно теперь называется «Очистка MacBook от мусора».
- С индикатора прогресса убрана отвлекающая бегущая светлая полоса. Процент и
  живое обновление хода проверки сохранены.
- У полного аудита больше нет общего ограничения времени. После поиска
  кандидатов плагин обрабатывает их все, пока пользователь явно не остановит
  проверку или не произойдёт реальная внутренняя ошибка.
- Отдельные системные команды macOS по-прежнему ограничены 30 секундами. Если
  одна команда не успела завершиться, плагин показывает предупреждение о
  неполноте данных для этого источника и продолжает аудит остальных кандидатов.
- Среда выполнения плагина использует точные версии `fast-uri@3.1.4` и
  `@hono/node-server@2.0.10`.

## Безопасность

Правила очистки не изменились. Плагин работает локально, не передаёт найденные
файлы в сеть и не показывает модели полные пути и секреты. Пробел покрытия
блокирует изменяющее действие для затронутого объекта. Перемещение,
восстановление и окончательное удаление относятся только к одному объекту и
требуют отдельного нажатия пользователя.

## Обновление с beta.9

Удалять источник плагинов или установленный плагин не нужно. В новой задаче
Codex выберите Codex Mac Cleaner и напишите одну команду:

> Обнови Codex Mac Cleaner до v0.1.0-beta.10.

После успешного обновления полностью завершите Codex, откройте его снова и
создайте новую задачу.

## Новая установка

```bash
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.10
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После перезапуска Codex создайте новую задачу, выберите Codex Mac Cleaner и
напишите: «Проверь Mac на остатки удалённых приложений и открой результаты».

## Ограничения предварительной версии

- Ручная проверка этой сборки на реальном Mac ещё не выполнена и остаётся
  отдельным этапом допуска владельца к выпуску.
- Продолжительность полного аудита заранее не ограничена и зависит от числа
  объектов, скорости диска и ответов macOS.
- Фоновая и автоматическая проверка в v0.1 отсутствует.
- Системные операции в `/Library`, `sudo`, управление службами, Homebrew и
  глобальными средами разработки не поддерживаются.
- Плагин пока не опубликован в общем каталоге плагинов Codex.

## Как проверить выпуск

К выпуску на GitHub должны быть приложены архив плагина, контрольная сумма
SHA-256, перечень файлов и зависимостей, сведения о происхождении сборки и
сторонних лицензиях. Повторная сборка из того же коммита должна создать тот же
архив.
