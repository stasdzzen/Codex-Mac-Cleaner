# Codex Mac Cleaner v0.1.0-beta.11

Версия beta.11 исправляет две причины, из-за которых полный аудит мог
завершиться, но не показать результат. Плагин теперь получает инструменты
штатным способом внутри Codex, использует точную завершённую ревизию проверки и
передаёт большой список находок в Dashboard небольшими безопасными страницами.

## Что изменилось

- Если инструменты плагина ещё не появились в текущей задаче, навык один раз
  запускает штатный поиск инструментов Codex. Он не обращается к Терминалу, не
  запускает MCP-процесс напрямую и не создаёт скрытый повторный аудит.
- Состояние проверки сообщает точный номер завершённой ревизии. Dashboard
  открывается только для этой ревизии и не использует догадки о номере.
- Большой результат больше не отправляется одним многомегабайтным сообщением.
  Модель и интерфейс получают не более 100 находок и 512 КиБ за один ответ.
- Dashboard v3 показывает общее число найденных объектов и загружает следующую
  страницу только после нажатия «Показать ещё».
- Страницы модели и интерфейса используют разные непрозрачные курсоры. Подмена
  курсора, ревизии, аудита или фильтра блокируется без раскрытия путей.
- Уже загруженные строки не исчезают при очередном обновлении состояния
  Dashboard.

## Безопасность

Правила очистки не ослаблены. Проверка выполняется локально, а полные пути,
пароли, токены и содержимое конфигураций не показываются модели. Плагин не
удаляет объекты автоматически и не предлагает массовые действия. Перемещение в
карантин, восстановление и окончательное удаление выполняются по одному объекту
и требуют отдельных нажатий пользователя.

## Обновление с beta.10

Удалять источник плагинов или установленный плагин не нужно. В новой задаче
Codex выберите Codex Mac Cleaner и напишите:

> Обнови Codex Mac Cleaner до v0.1.0-beta.11.

После успешного обновления полностью завершите Codex, откройте его снова и
создайте новую задачу. Уже открытая задача продолжает использовать прежнюю
версию.

## Новая установка

```bash
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.11
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После перезапуска Codex создайте новую задачу, выберите Codex Mac Cleaner и
напишите: «Проверь Mac на остатки удалённых приложений и открой результаты».

## Ограничения предварительной версии

- Ручная проверка beta.11 на реальном Mac ещё не выполнена и остаётся отдельным
  этапом владельца после публикации.
- Полный аудит не ограничен общим временем и может работать долго на Mac с
  большим числом объектов. Он должен завершиться, если пользователь сам его не
  остановил и не произошла реальная внутренняя ошибка.
- Фоновая и автоматическая проверка в v0.1 отсутствует.
- Системные операции в `/Library`, `sudo`, управление службами, Homebrew и
  глобальными средами разработки не поддерживаются.
- Плагин пока не опубликован в общем каталоге плагинов Codex.

## Как проверить выпуск

К выпуску на GitHub прилагаются архив плагина, SHA-256, контрольные суммы
файлов, SBOM, сведения о происхождении сборки и уведомления о сторонних
лицензиях. Повторная сборка из того же коммита должна создать тот же архив.
