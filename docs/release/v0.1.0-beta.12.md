# Codex Mac Cleaner v0.1.0-beta.12

Версия beta.12 делает диагностику плагина проверяемой до публикации. Это
технический выпуск без изменений логики поиска и очистки: правила безопасности,
классификация находок и поэлементные действия остались прежними.

## Что улучшено

- Перед выпуском автоматически проверяется точный набор инструментов, доступных
  модели и интерфейсу Codex.
- Отдельно проверяются адрес, формат и политика безопасности Dashboard v3.
- Ошибка запуска диагностической проверки показывает только безопасный код и
  не раскрывает путь к плагину или временной папке.
- Документация теперь объясняет, что делать, если инструменты не появились,
  ревизия результата устарела или Dashboard не открылся.
- Большие результаты по-прежнему загружаются ограниченными страницами, а
  повторный аудит не запускается скрытно.

## Безопасность

Плагин не удаляет найденные объекты автоматически и не предлагает массовые
действия. Полные пути, пароли, токены и содержимое конфигураций не передаются
модели. Карантин, восстановление и окончательное удаление выполняются только
для одного объекта после отдельного нажатия пользователя.

## Обновление с beta.11

Удалять источник плагинов или установленный плагин не нужно. В новой задаче
Codex выберите Codex Mac Cleaner и напишите:

> Обнови Codex Mac Cleaner до v0.1.0-beta.12.

После успешного обновления полностью завершите Codex, откройте его снова и
создайте новую задачу. Уже открытая задача продолжает использовать прежнюю
версию.

## Новая установка

```bash
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.12
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После перезапуска Codex создайте новую задачу, выберите Codex Mac Cleaner и
напишите: «Проверь Mac на остатки удалённых приложений и открой результаты».

## Ограничения предварительной версии

- Ручная проверка beta.12 на реальном Mac остаётся отдельным этапом владельца
  после публикации.
- Полный аудит не ограничен общим временем и может работать долго.
- Фоновая и автоматическая проверка в v0.1 отсутствует.
- Системные операции в `/Library`, `sudo`, управление службами, Homebrew и
  глобальными средами разработки не поддерживаются.
- Плагин пока не опубликован в общем каталоге плагинов Codex.

## Как проверить выпуск

К выпуску на GitHub прилагаются архив плагина, SHA-256, контрольные суммы
файлов, SBOM, provenance и уведомления о сторонних лицензиях. Повторная сборка
из того же коммита должна создать тот же архив.
