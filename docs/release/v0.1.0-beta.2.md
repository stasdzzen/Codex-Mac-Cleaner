# Codex Mac Cleaner v0.1.0-beta.2

Beta.2 восстанавливает запуск MCP-сервера после установки плагина из repository
marketplace. Codex снова может загрузить инструменты аудита и открыть Audit
Dashboard без копирования команд очистки в терминал.

## Исправления

* Исправлен packaged MCP-манифест: entrypoint и ресурсы теперь разрешаются от
  фактического корня установленного плагина, а не через неподдерживаемые
  буквальные placeholder.
* Добавлен clean-room regression test, который запускает скомпилированный
  runtime через публичный `.mcp.json`, выполняет MCP handshake, получает список
  инструментов и читает ресурс Dashboard.

## Обновление с beta.1

Marketplace beta закреплён на точном Git-теге. Чтобы перейти на beta.2, удалите
старую установку и source, затем добавьте marketplace заново:

```bash
codex plugin remove codex-mac-cleaner@codex-mac-cleaner
codex plugin marketplace remove codex-mac-cleaner
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.2
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После обновления полностью завершите Codex, откройте его снова и создайте новую
задачу. Попросите проверить остатки удалённых приложений и открыть Audit
Dashboard.

## Ограничения beta

* Ручной Real-Mac smoke владельца ещё не завершён; beta.2 выпускается именно
  для этой проверки и пока не считается готовой к работе с ценными данными.
* В v0.1 нет автоматического расписания, массовых действий, `/Library`, `sudo`,
  privileged helper, Homebrew upgrades и глобального управления SDK.
* Автоматически actionable могут быть только доказуемо связанные private
  cache/log remnants; остальные категории остаются inspect-only.
* Публикации в общей Plugin Directory пока нет.

## Целостность артефакта

GitHub prerelease содержит deterministic tar-архив и SHA-256. В архив включены
CycloneDX SBOM и provenance, связанные с commit release-тега. Ручной smoke
фиксируется отдельно и не объявляется выполненным в этих заметках.
