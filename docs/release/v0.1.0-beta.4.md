# Codex Mac Cleaner v0.1.0-beta.4

Beta.4 избавляет от ручного удаления установленного плагина и закладывает
обновление будущих версий одной явной фразой в чате Codex. Пользователь всегда
указывает точный опубликованный тег, а updater проверяет источник, результат и
восстановление предыдущей версии при ошибке.

## Новая возможность

* В пакет добавлен отдельный навык `codex-mac-cleaner-update`. После установки
  beta.4 будущий релиз можно запросить в новой задаче, например: «Обнови Codex
  Mac Cleaner до v0.1.0-beta.5».
* Updater принимает только точный release-тег и канонический GitHub repository.
  Он не выбирает `latest`, не меняет конфигурационные файлы напрямую и никогда
  не вызывает удаление установленного плагина.
* Если обновление marketplace или повторная установка не завершились, updater
  восстанавливает `v<предыдущая версия>` и проверяет результат. Неподтверждённый
  rollback возвращается как блокирующая ошибка.

## Обновление с beta.3

Beta.3 ещё не содержит update-навык, поэтому один раз нужно заменить только
закреплённый marketplace snapshot. Сам установленный плагин удалять не нужно:

```bash
codex plugin marketplace remove codex-mac-cleaner
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.4
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После обновления полностью завершите Codex, откройте его снова и создайте новую
задачу. Текущий процесс не может подменить уже загруженный набор навыков и
MCP-инструментов.

## Ограничения beta

* Ручной Real-Mac smoke владельца для этого release SHA ещё не завершён;
  сначала используйте beta только на некритичных данных.
* В v0.1 нет автоматического расписания, массовых действий, `/Library`, `sudo`,
  privileged helper, Homebrew upgrades и глобального управления SDK.
* Actionable могут быть только доказуемо связанные private cache/log remnants;
  остальные категории остаются inspect-only.
* Основной аудит остаётся локальным. Сеть используется только по явному
  запросу обновления для канонического GitHub marketplace.
* Публикации в общей Plugin Directory пока нет.

## Целостность артефакта

GitHub prerelease содержит deterministic tar-архив, SHA-256 и file checksums.
CycloneDX SBOM, provenance и third-party notices находятся внутри архива и
связаны с commit release-тега. Ручной smoke фиксируется отдельно и не
объявляется выполненным в этих заметках.
