# Codex Mac Cleaner v0.1.0-beta.5

Beta.5 снова показывает Audit Dashboard как визуальное приложение внутри
Codex. Предыдущая beta могла открыть панель, но вывести в ней минифицированный
JavaScript вместо интерфейса.

## Исправления

* Автономный Dashboard package теперь сохраняет JavaScript и CSS без
  повреждения служебными шаблонами строковой замены.
* Плагин передаёт текущие MCP Apps metadata и совместимый app-surface контракт,
  используемый Codex Security. Конкретное размещение панели остаётся решением
  Codex host.
* Регрессионная проверка требует один HTML document shell, один module script,
  один style block и отсутствие внешних script sources.

## Обновление с beta.4

Удалять marketplace или установленный плагин не нужно. В новой задаче Codex
выберите Codex Mac Cleaner и напишите:

> Обнови Codex Mac Cleaner до v0.1.0-beta.5.

После успешного обновления полностью завершите Codex, откройте его снова и
создайте новую задачу. Уже запущенный процесс не может заменить загруженный
набор навыков и MCP-инструментов.

## Новая установка

```bash
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.5
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После перезапуска Codex откройте новую задачу и напишите: «Проверь остатки
удалённых приложений и открой Audit Dashboard».

## Ограничения beta

* Ручной Real-Mac smoke владельца для release SHA ещё не завершён; сначала
  используйте beta только на некритичных данных.
* Физическое размещение app surface, включая правую панель, контролирует Codex
  host. Плагин передаёт тот же совместимый render-контракт, что Codex Security.
* В v0.1 нет автоматического расписания, массовых действий, `/Library`, `sudo`,
  privileged helper, Homebrew upgrades и глобального управления SDK.
* Actionable могут быть только доказуемо связанные private cache/log remnants;
  остальные категории остаются inspect-only.
* Публикации в общей Plugin Directory пока нет.

## Целостность артефакта

GitHub prerelease содержит deterministic tar-архив, SHA-256 и file checksums.
CycloneDX SBOM, provenance и third-party notices находятся внутри архива и
связаны с commit release-тега. Ручной smoke фиксируется отдельно и не
объявляется выполненным в этих заметках.
