# Codex Mac Cleaner v0.1.0-beta.6

Beta.6 делает длительный аудит заметно практичнее и даёт пользователю явное
управление доступными режимами отображения Dashboard.

## Улучшения

* **Управление Dashboard:** интерфейс остаётся встроенным в чат по умолчанию,
  но теперь пользователь может запросить полноэкранный режим или компактное
  окно, если текущая версия Codex поддерживает этот режим. Отказ host не
  прерывает аудит и не скрывает результаты.
* **Большие аудиты:** список установленных пакетов собирается один раз для
  каждой из двух проверочных фаз, а кандидаты анализируются с безопасно
  ограниченной параллельностью. Результаты по-прежнему показываются в
  детерминированном порядке.

## Исправления

* Исправлена причина, по которой реальный аудит с несколькими тысячами
  кандидатов мог остановиться по пятиминутному тайм-ауту до формирования
  результатов.
* Параллельные обновления истории владельцев больше не могут потерять уже
  подтверждённую запись.
* Отмена одного аудита останавливает все его активные проверки и не создаёт
  частичную actionable revision.

## Обновление с beta.5

Удалять marketplace или установленный плагин не нужно. В новой задаче Codex
выберите Codex Mac Cleaner и напишите:

> Обнови Codex Mac Cleaner до v0.1.0-beta.6.

После успешного обновления полностью завершите Codex, откройте его снова и
создайте новую задачу. Уже запущенный процесс не может заменить загруженный
набор навыков и MCP-инструментов.

## Новая установка

```bash
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.6
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После перезапуска Codex откройте новую задачу и напишите: «Проверь остатки
удалённых приложений и открой Audit Dashboard».

## Ограничения beta

* Ручной Real-Mac smoke владельца для release SHA ещё не завершён; сначала
  используйте beta только на некритичных данных.
* Codex host решает, поддерживаются ли fullscreen/PiP и где физически
  размещается app surface. Плагин не может гарантировать правую панель.
* Пятиминутный server-owned deadline сохранён. Timeout остаётся fail-closed и
  не создаёт находки, разрешённые к изменению.
* В v0.1 нет автоматического расписания, массовых действий, `/Library`, `sudo`,
  privileged helper, Homebrew upgrades и глобального управления SDK.
* Actionable могут быть только доказуемо связанные private cache/log remnants;
  остальные категории остаются inspect-only.
* Публикации в общей Plugin Directory пока нет.

## Целостность артефакта

GitHub prerelease содержит deterministic tar-архив, SHA-256 и file checksums.
CycloneDX SBOM, provenance и third-party notices находятся внутри архива и
связаны с commit release-тега. Ручной smoke фиксируется отдельно и не
объявляется выполненным в этих заметках.
