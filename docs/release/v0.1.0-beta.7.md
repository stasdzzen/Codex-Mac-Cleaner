# Codex Mac Cleaner v0.1.0-beta.7

Beta.7 исправляет остановку аудита на реальном Mac, делает результаты понятнее
и завершает согласованные элементы управления Audit Dashboard.

## Улучшения

* **Полезный Real-Mac аудит:** кандидаты анализируются с измеренной
  ограниченной параллельностью восемь при сохранении детерминированного порядка,
  Snapshot A/B и прежнего пятиминутного fail-closed deadline.
* **Читаемые размеры:** byte-метрики Dashboard отображаются в десятичных МБ и
  ГБ, включая корректное представление значений меньше 0,01 МБ.
* **Понятные компоненты:** локальный Dashboard может показывать очищенное имя
  верхнеуровневого компонента. Модель по-прежнему не получает полный путь,
  локальный app inventory, идентификаторы пакетов, токены или секреты.
* **Read-only диагностика:** отсутствующие targets пользовательских
  LaunchAgents, системных LaunchAgents/LaunchDaemons и активных процессов могут
  отображаться как диагностические находки. Они не получают действий очистки,
  отключения или завершения процесса.

## Dashboard

* Удалён режим «Мини-окно»/Picture-in-Picture; явный пользовательский запрос
  fullscreen сохранён.
* Добавлен подвал со ссылками на репозиторий, GitHub Ideas, разработчика и
  будущую страницу поддержки проекта.
* Переходы выполняются только после клика пользователя через host и ограничены
  разрешёнными доменами GitHub и Dzzen.

## Исправления

* Устранена причина двух подтверждённых тайм-аутов аудита на 2 044/2 766 и
  2 016/2 766 обработанных кандидатах.
* Добавлена детерминированная regression-модель на 2 766 кандидатов без
  увеличения deadline и без обрезки результатов.
* System/autostart/process диагностика остаётся строго read-only и не расширяет
  полномочия карантина или удаления.

## Обновление с beta.6

Удалять marketplace или установленный плагин не нужно. В новой задаче Codex
выберите Codex Mac Cleaner и напишите:

> Обнови Codex Mac Cleaner до v0.1.0-beta.7.

После успешного обновления полностью завершите Codex, откройте его снова и
создайте новую задачу. Уже запущенный процесс не может заменить загруженный
набор навыков и MCP-инструментов.

## Новая установка

```bash
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.7
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После перезапуска Codex откройте новую задачу и напишите: «Проверь остатки
удалённых приложений и открой Audit Dashboard».

## Ограничения beta

* Ручной Real-Mac smoke владельца для release SHA ещё не завершён; сначала
  используйте beta только на некритичных данных.
* Codex host управляет физическим размещением app surface. Плагин запрашивает
  fullscreen, но не может гарантировать конкретную правую панель.
* Пятиминутный server-owned deadline сохранён. Timeout остаётся fail-closed и
  не создаёт находки, разрешённые к изменению.
* В v0.1 нет автоматического расписания, массовых действий, `/Library`, `sudo`,
  privileged helper, Homebrew upgrades и глобального управления SDK.
* Actionable могут быть только доказуемо связанные private cache/log remnants;
  остальные категории остаются inspect-only.
* Публикации в общей Plugin Directory пока нет.

## Целостность артефакта

GitHub prerelease содержит deterministic tar-архив, SHA-256 и file checksums.
CycloneDX SBOM, provenance и third-party notices связаны с commit release-тега
и доступны как отдельные assets и внутри архива. Ручной smoke фиксируется
отдельно и не объявляется выполненным в этих заметках.
