# Codex Mac Cleaner v0.1.0-beta.8

Версия beta.8 обновляет окно проверки, использует официальные компоненты
shadcn для Base UI и объясняет результаты без внутренних кодов плагина.

## Что изменилось

- Все кнопки, вкладки, таблицы и окна подтверждения используют компоненты
  shadcn для Base UI. Прежняя UI-реализация и отменённый дизайн-эксперимент
  удалены.
- Светлая и тёмная темы используют согласованную палитру Base Mira.
- Компоновка адаптируется к узкому окну внутри Codex и к развёрнутому режиму.
- Окно проверки, описание плагина, его команды и README переписаны понятным русским
  языком.
- Внутренние коды классификации, риска, доказательств и причин блокировки
  преобразуются в безопасные русские объяснения.
- Логотип проекта отображается в окне проверки, карточке плагина и README.

## Безопасность

Границы безопасности не изменились. Плагин выполняет локальную проверку без
телеметрии, не показывает модели полные пути и секреты, не удаляет объекты
автоматически и не выполняет массовые действия. Перемещение, восстановление и
окончательное удаление относятся только к одному объекту и подтверждаются
отдельно.

## Обновление предыдущей версии

Удалять источник плагинов или установленный плагин не нужно. В новой задаче
Codex выберите Codex Mac Cleaner и напишите:

> Обнови Codex Mac Cleaner до v0.1.0-beta.8.

После успешного обновления полностью завершите Codex, откройте его снова и
создайте новую задачу.

## Новая установка

```bash
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.8
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После перезапуска Codex создайте новую задачу, выберите Codex Mac Cleaner и
напишите: «Проверь Mac на остатки удалённых приложений и открой результаты».

## Ограничения предварительной версии

- Владелец проекта ещё должен вручную проверить эту сборку на реальном Mac.
- Фоновая и автоматическая проверка в v0.1 отсутствует.
- Системные операции в `/Library`, `sudo`, управление службами, Homebrew и
  глобальными SDK не поддерживаются.
- Плагин может запросить развёрнутый режим, но размещением окна управляет Codex.
- Плагин пока не опубликован в общем каталоге плагинов Codex.

## Целостность выпуска

К предварительному выпуску на GitHub приложены архив плагина, его контрольная
сумма SHA-256, перечень файлов и зависимостей, сведения о происхождении сборки
и сторонних лицензиях. Повторная сборка из того же коммита создаёт тот же
архив, поэтому его содержимое можно проверить.
