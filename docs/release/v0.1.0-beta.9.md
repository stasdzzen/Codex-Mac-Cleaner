# Codex Mac Cleaner v0.1.0-beta.9

Версия beta.9 делает карточку плагина, окно проверки, сообщения Codex, README
и публичные материалы GitHub понятными обычному владельцу Mac.

## Что изменилось

- В карточке плагина имя разработчика и описания написаны по-русски.
- Основные действия используют единые названия: «Оставить», «В карантин»,
  «Восстановить» и «Удалить навсегда».
- В окне проверки больше нет необъяснённых слов `Dashboard`, `snapshot`,
  `evidence`, `preview` и других внутренних терминов.
- Названия инструментов и сообщения Codex объясняют проверку простыми словами.
- README, документы поддержки и безопасности, формы GitHub и заметки о выпуске
  отредактированы на русском языке.

## Безопасность

Правила безопасности не изменились. Плагин проверяет Mac локально, не передаёт
найденные файлы в сеть, не показывает модели полные пути и секреты и ничего не
удаляет автоматически. Перемещение, восстановление и окончательное удаление
относятся только к одному объекту и требуют отдельного нажатия пользователя.

## Обновление с beta.8

Удалять источник плагинов или установленный плагин не нужно. В новой задаче
Codex выберите Codex Mac Cleaner и напишите:

> Обнови Codex Mac Cleaner до v0.1.0-beta.9.

После успешного обновления полностью завершите Codex, откройте его снова и
создайте новую задачу.

## Новая установка

```bash
codex plugin marketplace add stasdzzen/Codex-Mac-Cleaner --ref v0.1.0-beta.9
codex plugin add codex-mac-cleaner@codex-mac-cleaner
```

После перезапуска Codex создайте новую задачу, выберите Codex Mac Cleaner и
напишите: «Проверь Mac на остатки удалённых приложений и открой результаты».

## Ограничения предварительной версии

- Владелец проекта ещё должен вручную проверить эту сборку на реальном Mac.
- Фоновая и автоматическая проверка в v0.1 отсутствует.
- Системные операции в `/Library`, `sudo`, управление службами, Homebrew и
  глобальными средами разработки не поддерживаются.
- Плагин пока не опубликован в общем каталоге плагинов Codex.

## Как проверить выпуск

К выпуску на GitHub приложены архив плагина, контрольная сумма SHA-256,
перечень файлов и зависимостей, сведения о происхождении сборки и сторонних
лицензиях. Повторная сборка из того же коммита создаёт тот же архив.
