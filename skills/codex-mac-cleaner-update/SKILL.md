# Обновление Codex Mac Cleaner

Используй этот навык только после явного запроса пользователя обновить Codex
Mac Cleaner до точного тега вида `vX.Y.Z`, `vX.Y.Z-beta.N`,
`vX.Y.Z-alpha.N` или `vX.Y.Z-rc.N`.

1. Повтори точный целевой тег и объясни, что update обращается только к
   каноническому GitHub repository; данные аудита и пользовательские файлы не
   отправляются. Не выбирай `latest` и не ищи версию за пользователя.
2. Определи корень установленного плагина относительно этого `SKILL.md`; не
   угадывай путь к cache и не используй путь репозитория разработчика.
3. Запусти `node <корень-плагина>/scripts/codex-mac-cleaner-update.mjs --json
   <точный-тег>`.
4. При `updated` сообщи версии до и после, затем попроси полностью завершить
   Codex, открыть его снова и создать новую задачу.
5. При `already_current` сообщи, что обновление не требуется.
6. При `failed` покажи только безопасный `errorCode` и состояние `rollback`.
   Если rollback имеет значение `failed`, остановись и не предлагай обход.

Updater заменяет только закреплённый Git ref marketplace через Codex CLI и
повторно устанавливает cache плагина. Он не вызывает `codex plugin remove`, не
редактирует `config.toml` или marketplace-файлы напрямую, не использует
инструменты аудита/карантина и не касается пользовательских файлов.
