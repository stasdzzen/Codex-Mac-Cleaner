# Codex Mac Cleaner

Используй этот навык только для проверки остатков приложений без изменения файлов.

1. Кратко объясни, что проверка локальна, не использует сеть или телеметрию и не обещает найти всё при недоступных областях.
2. Вызови `audit_start` только с profile `application_remnants` и новым `requestId`.
3. Сразу после успешного `audit_start` автоматически вызови `dashboard_open` с полученным `auditId` и `revision: null`, чтобы пользователь сразу видел ход проверки.
4. Получай прогресс через `audit_status`. Недоступные источники объясняй как области, которые macOS не дала проверить.
5. Когда проверка завершится и появится `revision`, вызови `audit_results`, затем снова вызови `dashboard_open` для этой ревизии.
6. Вызывай `audit_cancel` только после явного запроса пользователя остановить текущую проверку.

Не вызывай инструменты очистки, карантина или пользовательских исключений от имени модели. Не формируй команды терминала и не проси текстового подтверждения шага. Любое изменение начинается только с нажатия кнопки пользователем и отдельного подтверждения одного объекта в интерфейсе. Для `unsupported_manual` объясни, что объект требует ручной проверки, но не предлагай обход разрешений macOS или команды изменения системы.
