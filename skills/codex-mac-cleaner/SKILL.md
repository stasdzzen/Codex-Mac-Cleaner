# Codex Mac Cleaner

Используй этот навык только для проверки остатков приложений без изменения файлов.

Используй только инструменты Codex Mac Cleaner, которые штатно предоставил host текущей задаче. Если `audit_start`, `audit_status`, `audit_results` или `dashboard_open` не видны сразу, один раз выполни штатный поиск инструментов Codex по этим точным именам. Если после поиска хотя бы один из них недоступен, остановись и прямо сообщи, что Codex не загрузил инструменты плагина. Не создавай и не запускай локальные скрипты, не подключай MCP напрямую через stdio, не открывай локальный HTML и не заявляй, что Dashboard открыт.

1. Кратко объясни, что проверка локальна, не использует сеть или телеметрию и не обещает найти всё при недоступных областях.
2. Вызови `audit_start` только с profile `application_remnants` и новым `requestId`.
3. Сразу после успешного `audit_start` автоматически вызови `dashboard_open` с полученным `auditId` и `revision: null`, чтобы пользователь сразу видел ход проверки.
4. Получай прогресс через `audit_status`. До состояний `completed` или `completed_with_warnings` поле `revision` обязано оставаться `null`. Недоступные источники объясняй как области, которые macOS не дала проверить.
5. После успешного завершения возьми точное целое значение `revision` только из ответа `audit_status`. С этим же значением вызови `audit_results`, затем `dashboard_open`. Не подставляй номер ревизии самостоятельно.
6. Вызывай `audit_cancel` только после явного запроса пользователя остановить текущую проверку.

Если сервер вернул `AUDIT_STALE` или успешное завершение без целого `revision`, остановись: результаты недоступны. Не запускай новый или повторный аудит автоматически — дождись явного запроса пользователя.

Не вызывай инструменты очистки, карантина или пользовательских исключений от имени модели. Не формируй команды терминала и не проси текстового подтверждения шага. Любое изменение начинается только с нажатия кнопки пользователем и отдельного подтверждения одного объекта в интерфейсе. Для `unsupported_manual` объясни, что объект требует ручной проверки, но не предлагай обход разрешений macOS или команды изменения системы.
